# projeto-lury
■ README – Sistema Web de Coleta
PROESP-Br (Simulado)
■ Visão Geral
Este projeto é um protótipo funcional de uma interface web para coleta de dados
baseada no Manual PROESP-Br. O sistema segue princípios fundamentais de
IHM (Interação Humano■Computador), priorizando: Eficiência Prevenção de
erros Feedback imediato Acessibilidade Ele é composto por 3 etapas (Triagem,
Antropometria e Testes Motores), conforme a Lei de Hick■Hyman. Todos os
dados são simulados (Opção A) e armazenados por localStorage.
■ Estrutura dos Arquivos
index.html – Triagem antropometria.html – Antropometria testesmotores.html
– Testes Motores scripts.js – Lógica, validações e navegação style.css – Estilo
visual e acessibilidade
■ Recursos Implementados
1. Feedback Imediato: Mensagens de erro e sucesso Campos destacados
dinamicamente 2. Prevenção de Erros: Validação por faixa e tipo Bloqueio de
avanço quando necessário 3. Gestalt Aplicada: Agrupamento visual por blocos
Hierarquia clara de informações 4. Lei de Fitts: Botões maiores Alvos de clique
amplos 5. Acessibilidade: Contraste adequado Tab navega por todos os
campos
■ Lógica do Sistema
Validação automática ao sair dos campos Dados persistidos em localStorage
Navegação entre páginas só ocorre se válido Relatório final totalmente simulado
■ Tecnologias Usadas
HTML5 semântico CSS3 responsivo JavaScript Puro (Vanilla)
✔ Conclusão
O sistema demonstra: Aplicação correta dos princípios de IHM Navegação clara
e estruturada Feedback constante e preventivo Simulação completa e funcional
(Opção A) O protótipo é adequado para fins educacionais e demonstra boa
prática de design e interação.
